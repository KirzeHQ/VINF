# Contributing to VINF

Thanks for your interest in wanting to contribute!  
This is a research project in early development, so the best way to contribute is to:
- Implement the algorithms in `docs/` (see `VINF-HASH.md` and `VINF-HASH-CUSTOM.md` for the hashing algorithm)
- Add unit tests and examples for the algorithms you implement
- Provide feedback on the design and spec in `docs/`
- Report any issues you find in the code or spec
- Suggest improvements or optimizations to the algorithms
- Do audits or security reviews of the algorithms (keep in mind these are custom constructions without formal guarantees, so be critical and cautious)
- Help with documentation, formatting, and clarity of the spec and code

## Guidelines

- Follow the existing code style and conventions in the project
- Write clear commit messages that explain the purpose of your changes
- If you're implementing an algorithm from the spec, reference the relevant section in your code comments
- If you're adding tests, try to cover edge cases and provide explanations for what the test is
- Be respectful and constructive in your feedback and discussions
- If you're unsure about something in the spec or implementation, ask for clarification before making assumptions
- Keep in mind that this is an experimental research project, so there may be changes to the spec and algorithms as we learn and iterate. Be flexible and open to changes.

## How to contribute

1. Fork the repo and either make a new branch or just use the main branch for your changes.
2. Implement your code changes or improvements.
3. Add tests and documentation for your changes. (if any)
4. Commit your changes with a clear message.
5. Open a PR with a description of your changes and how they fit into the project.
6. Give me (or anyone else with write access) a day or two to review it and provide feedback.
7. Address any feedback and make necessary changes.
8. Once it's approved, it will be merged into the main branch!
